# План реализации MVP

**Версия:** 1.0
**Дата:** март 2026

> Последовательный список сторей для реализации MVP голосового AI-агента. Каждая стори строится на результатах предыдущей. Описание продукта — в [about.md](about.md), архитектура — в [architecture.md](architecture.md), спецификации — в [specs.md](specs.md).

---

- [ ] **S01. Docker-инфраструктура и заглушки сервисов**
  Развертка monorepo-структуры проекта и Docker Compose с семью контейнерами-заглушками: `caddy`, `api`, `livekit`, `agent`, `litellm`, `postgres`, `coturn`. Каждый контейнер стартует и отвечает health check, но не содержит бизнес-логики. Настроены внутренняя сеть `twype-net`, volumes, `.env.example`, конфиги (`livekit.yaml`, `litellm.yaml`, `Caddyfile`, `turnserver.conf`). Dev- и production-композы. Корневые `pyproject.toml` (uv workspace) и `package.json` (bun workspace). Линтинг: ruff для Python, ESLint + Prettier для TypeScript.

- [ ] **S02. Схема базы данных и миграции**
  SQLAlchemy 2.0 модели (async, Mapped columns): `users`, `sessions`, `messages`, `agent_configs`, `agent_prompts`, `knowledge_chunks`, `verification_codes`, `crisis_contacts`. Расширение pgvector. Alembic-инфраструктура в `apps/api/migrations/`. Начальная миграция, seed-скрипт с тестовыми данными. Соглашения по именованию таблиц и индексов.

- [ ] **S03. Аутентификация (регистрация, логин, JWT)**
  FastAPI-эндпоинты: `POST /auth/register`, `POST /auth/verify`, `POST /auth/login`, `POST /auth/refresh`. Регистрация с email + password, хеширование через bcrypt/passlib, отправка 6-значного кода через Resend, верификация email. JWT-токены: access (15 мин, HS256) + refresh (30 дней). Middleware для валидации Bearer-токенов. Тесты на все эндпоинты аутентификации.

- [ ] **S04. Управление сессиями и генерация LiveKit-токенов**
  FastAPI-эндпоинты: `POST /sessions/start` (создание сессии + генерация LiveKit-токена с правами), `GET /sessions/history` (список прошлых сессий пользователя), `GET /sessions/{id}/messages` (история сообщений сессии). Генерация LiveKit-токенов через `livekit-api` SDK с правами `can_publish`, `can_subscribe`, `can_publish_data`. Запись сессий в БД.

- [ ] **S05. LiveKit Agent: подключение к комнате**
  Базовый LiveKit Agent в `apps/agent/`: точка входа `main.py`, подключение к LiveKit Server через SDK, приём и диспатч job при создании комнаты. Агент подключается как участник к комнате, получает аудиопоток. Silero VAD для детекции речи. На этом этапе агент только слушает — без STT/LLM/TTS.

- [ ] **S06. Интеграция STT (Deepgram)**
  Подключение плагина `livekit-plugins-deepgram`. Стриминговое распознавание речи с промежуточными и финальными транскриптами. Поддержка русского и английского языков. Извлечение sentiment score (-1..1) из результатов Deepgram. Сохранение транскриптов пользователя в БД (`messages`). Отправка промежуточных транскриптов клиенту через data channel.

- [ ] **S07. Интеграция LLM через LiteLLM Proxy**
  Настройка LiteLLM Proxy: конфигурация Gemini Flash-Lite как основной модели и GPT-4.1-mini как fallback. Подключение OpenAI-совместимого плагина `livekit-plugins-openai` к LiteLLM. Базовый промпт-контекст (пока хардкод). Стриминговая генерация ответов. Health check LiteLLM и обработка недоступности. Сохранение ответов агента в БД.

- [ ] **S08. Интеграция TTS (Inworld)**
  Кастомный TTS-плагин для Inworld AI в `apps/agent/src/plugins/`. Стриминговый синтез речи из текста LLM-ответа. Поддержка русского и английского языков. Параметры экспрессивности и скорости. Fallback на ElevenLabs при недоступности Inworld. Плагин разрабатывается как модуль, пригодный для PR в репозиторий LiveKit Agents.

- [ ] **S09. Voice pipeline end-to-end**
  Сборка полного голосового pipeline: VAD -> Turn Detector -> STT -> LLM -> TTS -> WebRTC аудио. Настройка Turn Detector (определение конца реплики): пороги пауз, страховочный таймаут 3 сек. Шумоподавление на входе. Сквозной стриминг между всеми компонентами. Целевая задержка voice-to-voice ~800 мс. Thinking sounds и TTS-филлеры при длительной обработке.

- [ ] **S10. Система промптов из БД**
  Загрузка всех слоёв промптов из PostgreSQL при инициализации агента: системный промпт, промпт голосового режима, двухслойного ответа, эмоциональной адаптации, кризисного протокола, RAG-контекста, языковой адаптации, проактивных реплик. Prompt Builder формирует итоговый контекст для LLM. Config snapshot при старте сессии — фиксация версии конфигурации. Seed-скрипт с начальным набором промптов.

- [ ] **S11. Текстовый чат через data channel**
  Data Channel Handler в агенте: приём текстовых сообщений от клиента через LiveKit data channel. Маршрутизация текста напрямую в LLM (минуя STT). Стриминг текстового ответа обратно через data channel. Сохранение текстовых сообщений в ту же таблицу `messages` с меткой режима. При текстовом режиме TTS не вызывается.

- [ ] **S12. Переключение режимов: голос <-> текст**
  Единый поток диалога при переключении между режимами. Агент определяет текущий режим по типу входящих данных (аудиотрек vs data channel). Context Manager поддерживает общую историю для обоих режимов. При переключении на текст — LLM генерирует развёрнутые ответы; при переключении на голос — краткие, разговорные.

- [ ] **S13. RAG: загрузка базы знаний (Ingestion)**
  Скрипт `scripts/ingest.py`: извлечение текста из PDF, EPUB, DOCX, аудио (транскрибация). Семантическое чанкирование по смысловым блокам. Обогащение метаданными (source_type, title, author, url, section, page_range, language, tags). Генерация эмбеддингов через LiteLLM. Загрузка в PostgreSQL: vector-столбец + метаданные. Создание HNSW-индекса и tsvector для полнотекстового поиска.

- [ ] **S14. RAG: гибридный поиск при запросе**
  RAG Engine в агенте: преобразование реплики в эмбеддинг, гибридный поиск (cosine distance pgvector + tsvector полнотекстовый + фильтры по метаданным). Top-K (3-5) релевантных фрагментов включаются в контекст LLM вместе с метаданными источников. Приоритизация фрагментов на языке пользователя, но без исключения кросс-языковых результатов.

- [ ] **S15. Двухслойный ответ (голос + текст)**
  LLM формирует ответ в двух частях согласно промпту двухслойного ответа. Голосовая часть: 2-5 фраз, разговорный стиль, устные ссылки на источники. Текстовая часть: структурированные тезисы с массивами chunk_ids для каждого тезиса. Агент отправляет обе части: голосовую в TTS, текстовую через data channel. Маркировка ответов без подтверждения из базы знаний как рассуждений.

- [ ] **S16. Атрибуция источников (API + клиент)**
  FastAPI-эндпоинт `GET /sources/{chunk_ids}` — возвращает полные метаданные RAG-фрагментов (source_type, title, author, section, page_range, url). Клиент рендерит иконки-индикаторы (книга, видео, подкаст, статья) рядом с каждым тезисом. Попап при клике: автор, название, раздел, страница/таймкод, прямая ссылка.

- [ ] **S17. Эмоциональная адаптация (Circumplex)**
  Emotional Analyzer: приём sentiment score от Deepgram как быстрого сигнала. LLM-интерпретация состояния в двумерном пространстве (valence + arousal) на основе текста, sentiment, контекста и тренда. Адаптация тона и стиля ответа по квадрантам: паника, апатия, энтузиазм, спокойствие. Сохранение эмоциональных данных в БД для анализа тренда сессии.

- [ ] **S18. Проактивное поведение**
  Silence Timer в агенте: таймер тишины после ответа. При коротком таймауте (15 сек) — уточняющий вопрос по контексту. При длинном таймауте (45 сек) — мягкая реплика с учётом эмоционального состояния. Фразы генерирует LLM с флагом `proactive` / `extended_silence`, а не хардкод. Таймер сбрасывается при любой активности. Не чаще одной проактивной реплики за паузу.

- [ ] **S19. Кризисный протокол**
  Crisis Detector: анализ реплик на тревожные сигналы (суицид, самоповреждение, острые симптомы, насилие). При обнаружении — перехват pipeline, фиксированный протокол ответа: эмпатия, безоценочность, рекомендация профессиональной помощи, контакты экстренных служб. Контакты определяются по языку/локали из БД (таблица `crisis_contacts`). Логирование срабатываний. Высший приоритет — не переопределяется контекстом.

- [ ] **S20. Обработка перебиваний**
  Interruption handling: при обнаружении речи во время ответа агента — немедленная отмена текущей генерации LLM + TTS, переключение на приём нового ввода. Обработка ложных перебиваний: если после прерывания не распознано слов — перегенерация краткого продолжения или повтор последних 1-2 предложений. Настраиваемые таймауты.

- [ ] **S21. PWA: экраны аутентификации**
  React-приложение в `apps/web/`: страницы регистрации (email + password), верификации (ввод 6-значного кода), логина. React Router для маршрутизации. TanStack Query для запросов к FastAPI. Хранение JWT-токенов, автоматический refresh при 401. Редирект на логин при истечении refresh-токена. Базовый layout приложения.

- [ ] **S22. PWA: основной интерфейс чата**
  Подключение к LiveKit через Client SDK. Единый интерфейс голосового и текстового чата. Управление микрофоном (AgentControlBar). Лента чата с транскриптами (AgentChatTranscript). Визуализация аудио. Индикатор состояния агента. Поле ввода текста. Переключение между голосовым и текстовым режимами. Отображение промежуточных транскриптов в реальном времени.

- [ ] **S23. PWA: атрибуция источников и история**
  Рендеринг иконок-индикаторов источников (книга, видео, подкаст, статья) рядом с тезисами в текстовом чате. Попап с детальной информацией при клике (запрос к `GET /sources/{ids}`). Экран истории сессий (запрос к `GET /sessions/history`). Просмотр прошлых диалогов (`GET /sessions/{id}/messages`).

- [ ] **S24. PWA как Progressive Web App**
  Service worker для офлайн-shell. Web App Manifest (иконки, тема, standalone display). Мета-теги для мобильных. Адаптивная верстка для мобильных и десктопных браузеров. Оптимизация Vite-билда для production.

- [ ] **S25. Интеграционное тестирование и стабилизация**
  End-to-end проверка всех user flows: регистрация -> логин -> начало сессии -> голосовой диалог -> переключение на текст -> атрибуция источников -> проактивная реплика -> завершение. Тестирование на двух языках (русский, английский). Проверка кризисного протокола. Нагрузочный тест на 30 одновременных сессий. Исправление багов, тюнинг таймаутов и порогов. Финализация seed-данных и промптов.
