## Why

After the agent responds, the user may go silent — hesitating, thinking, or losing focus. Without proactive follow-up the conversation stalls and feels passive. A silence timer with two escalation stages (short follow-up question, longer gentle nudge) keeps the session alive and demonstrates attentiveness, especially valuable when emotional adaptation (S17) is active and can inform the tone of the nudge.

## What Changes

- New `SilenceTimer` module in the agent: an async timer that starts after each agent response and resets on any user activity (voice transcript, text message, VAD speech start)
- Two-stage timeout logic: short timeout (configurable, default 15 s) triggers a context-aware follow-up question; long timeout (configurable, default 45 s) triggers a gentle utterance adapted to the current emotional state
- Proactive utterances are generated by the LLM via `session.generate_reply` with a dedicated `proactive` or `extended_silence` flag injected into the prompt — no hardcoded phrases
- Guard: at most one proactive utterance per silence period; after firing, the timer stops until the next user activity restarts it
- New data channel message type `proactive_nudge` so the client knows the response was agent-initiated
- New settings: `PROACTIVE_ENABLED`, `PROACTIVE_SHORT_TIMEOUT`, `PROACTIVE_LONG_TIMEOUT`

## Capabilities

### New Capabilities
- `proactive-silence-timer`: Async silence timer with two-stage escalation, LLM-generated follow-ups, activity reset, and single-fire guard

### Modified Capabilities
- `agent-entrypoint`: Wire silence timer lifecycle (start/reset/stop) into existing event handlers in `main.py`
- `database-seed`: Add `proactive_prompt` seed content with `{proactive_type}` and `{emotional_context}` template placeholders for en locale
